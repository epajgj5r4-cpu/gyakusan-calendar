# 逆算カレンダー 仕様書

最終更新日: 2026-06-26
バージョン: 0.1（ドラフト）

---

## 1. 概要

### 1.1 これは何か
「逆算カレンダー」は、**目標達成日（ゴール日）から逆算して「あと何日」を毎日カウントダウン**していくWebアプリである。
通常のカレンダーが「今日が何日か」を示すのに対し、本アプリは「**目標まで残りどれだけか**」を主役にする。

### 1.1.1 設計コンセプト（最重要）
本アプリの本質は **「ユーザーを焦らせて行動を促す」** ことにある。
単なる進捗の記録・管理ツールではなく、**残り時間を突きつけ、危機感・緊張感を生むこと**が最優先の価値である。
したがって、すべてのUI・演出はこの「焦らせる」目的に奉仕するよう設計する。

#### 焦らせるための演出方針
- **残り時間を大きく・強く見せる**（画面の主役は残り時間の数字）。
- **時・分・秒までリアルタイムで減らす**（「今この瞬間も時間が減っている」感覚を出す）。
- **残りが少なくなるほど視覚的に強調する**（例: 色が緑→黄→赤に変化、数字が大きく/点滅、警告的なメッセージ）。
- **経過した時間は取り戻せないことを可視化する**（経過バー＝消費済みを強調）。
- ネガティブに不快なだけにならないよう、行動を促す前向きなメッセージも添える。

### 1.2 解決したい課題
- 目標までの残り時間が感覚的になり、行動が後回しになる
- 「いつまでに何をやるか」を日単位で見える化したい
- 1日ごとに減っていく残り日数を見て、緊張感とモチベーションを保ちたい

### 1.3 想定ユースケース
- 受験・試験まであと◯日
- 資格取得・プロジェクト納期まであと◯日
- イベント・大会・発表まであと◯日

## 2. スコープ（今回作るもの）

|------|------|
| 形態 | **Webアプリ（ブラウザ）** |
| 管理する目標数 | **1つのみ** |
| 主機能 | カウントダウン表示 ＋ **日々のタスク／メモ** |

- 進捗グラフ・統計（必要になれば追加）

## 3. 機能要件

### 3.1 目標設定
- 初回起動時、目標が未設定なら「目標設定画面」を表示する。
- 入力項目:
  - **目標名**（例: 「基本情報技術者試験」）必須
  - **開始日**（デフォルト: 今日）必須
  - **ゴール日**（達成したい日）必須・開始日より後であること
- 設定後はメイン画面（カウントダウン）へ遷移する。
- 設定はいつでも編集・リセットできる。

#### バリデーション
- ゴール日が開始日以前 → エラー表示して保存させない
- 目標名が空 → エラー表示

### 3.2 カウントダウン表示（メイン画面）※本アプリの主役
- 画面中央に **残り時間**を最も大きく表示する。
  - メイン表示は **残り日数**（例: `あと 42 日`）。
  - その直下に **時・分・秒**をリアルタイム表示し、毎秒減らす（例: `12時間 34分 56秒`）。
- 補助情報として表示する:
  - 目標名
  - ゴール日（例: 2026/08/07）
  - 経過日数 / 全体日数（例: 8 日経過 / 50 日中）
  - 進捗率（経過率）をバーで表示（経過＝消費済みを強調、例: 全体の 16% 消化）
- 残り時間の計算は **「ゴール日時 − 現在時刻」**を都度算出する（保存値ではない）。
- 当日（残り0日）: 「**今日が目標日です！**」を表示。
- ゴール日経過後: 「**目標日を過ぎました（◯日経過）**」を表示。

#### 「焦らせる」演出（緊急度ステージ）
残り日数に応じて画面の色調・メッセージを段階的に変える。閾値は仮置きで、実装時に調整可能とする。

| ステージ | 残り日数（目安） | 配色 | メッセージ例 |
|----------|------------------|------|--------------|
| 余裕 | 31日以上 | 青／緑 | 「今のうちに計画を進めよう」 |
| 注意 | 8〜30日 | 黄 | 「そろそろ本気を出す時期です」 |
| 警告 | 1〜7日 | オレンジ | 「ラストスパート！残りわずか」 |
| 緊急 | 当日 | 赤（点滅/強調） | 「ついに当日！今すぐ動こう」 |

- 緊急度が上がるほど数字を大きく／コントラストを強くするなど、視覚的圧を高める。

#### 演出強度（確定）
本バージョンの演出は以下の範囲とする（効果音・振動は**行わない**）。
- 配色の段階変化（緑→黄→オレンジ→赤）
- 緊急度に応じたメッセージの切り替え
- **数字の点滅**（緊急ステージで残り時間の数字を点滅させる）
- **数字の拡大**（緊急度が上がるほど残り時間のフォントを大きくする）
- **アニメーション**（秒数のカウント更新時に動きをつける／プログレスバーの変化を滑らかにする等）

#### 残り時間の定義
- メインの「日数」は時刻を切り捨てた**日付の差**で計算する（例: 今日が 6/26、ゴールが 6/28 なら「あと2日」）。
- 時・分・秒は**現在時刻からゴール日時（既定はゴール日の 00:00、設定で変更可）までの差**をリアルタイム計算する。
- 端末のローカル時刻を基準とする。

### 3.3 日々のタスク／メモ
- 各日付に対して **タスク（チェックリスト）** と **メモ（自由記述）**を記録できる。
- メイン画面に「今日のタスク／メモ」欄を表示する。
- タスク: テキスト追加・完了チェック・削除ができる。
- メモ: 自由記述テキスト（複数行）。
- 過去・未来の日付も選んで編集できる（簡易なカレンダー／日付リストから選択）。
- データは日付をキーに保存する。

### 3.4 リセット／編集
- 目標を編集（目標名・日付の変更）できる。
- 「目標をリセット」で全データ（目標＋タスク・メモ）を削除し初期状態に戻す（確認ダイアログあり）。

## 4. 非機能要件
- **オフライン動作**: ネット接続なしで動く（静的ファイル＋localStorage）。
- **レスポンシブ**: PC・スマホ両方のブラウザで見やすい。
- **データ永続化**: ブラウザを閉じても保持（localStorage）。同一ブラウザ・同一端末でのみ保持される点に注意。
- **軽量**: ページ読み込みが速い。フレームワーク不要レベルの構成を想定。

## 5. データ設計（localStorage）

```jsonc
{
  "goal": {
    "name": "基本情報技術者試験",
    "startDate": "2026-06-26",   // YYYY-MM-DD
    "goalDate":  "2026-08-07"    // YYYY-MM-DD
  },
  "days": {
    "2026-06-26": {
      "tasks": [
        { "id": "t1", "text": "過去問1年分", "done": true },
        { "id": "t2", "text": "苦手分野の復習", "done": false }
      ],
      "memo": "今日は調子が良かった"
    },
    "2026-06-27": { "tasks": [], "memo": "" }
  }
}
```

- 残り日数・経過日数・進捗率は**保存しない**（毎回計算する派生値）。

### 主要な計算式
- 全体日数 `totalDays = goalDate − startDate`
- 経過日数 `elapsedDays = today − startDate`（0未満は0に丸め）
- 残り日数 `remainingDays = goalDate − today`
- 進捗率 `progress = elapsedDays / totalDays`（0〜100%にクランプ）

## 6. 画面構成

### 6.1 画面一覧
1. **目標設定画面**（初回／編集時）
   - 目標名・開始日・ゴール日の入力フォーム、保存ボタン
2. **メイン画面（カウントダウン）**
   - 残り日数（大）、目標名、ゴール日、経過/全体、進捗バー
   - 今日のタスク・メモ欄
   - 日付選択（過去・未来の日に切り替え）
   - 目標編集／リセットへの導線

### 6.2 画面遷移
```
[初回起動] → 目標未設定？
   ├─ Yes → 目標設定画面 → (保存) → メイン画面
   └─ No  → メイン画面
メイン画面 ⇄ 目標設定画面（編集）
メイン画面 → リセット → 初回起動状態へ
```

## 7. 技術構成・デプロイ

### 7.1 技術構成
- **フロントのみ**の静的Webアプリ
  - HTML / CSS / JavaScript（バニラ、もしくは軽量構成）
  - 保存: `localStorage`
- ビルド不要で `index.html` を開けば動く構成を第一候補とする。

### 7.2 デプロイ方針（確定）
- **GitHub リポジトリ → Vercel で公開**する。
  - GitHub に push すると Vercel が自動でデプロイ（本番反映）する。
  - 静的サイトのため**ビルド設定不要**（Framework Preset: Other / Output: ルートの `index.html`）。
- ローカル開発: `index.html` をブラウザで開く、または簡易サーバーで確認。
- リポジトリ構成（案）:
  ```
  逆算カレンダー/
  ├── index.html        # アプリ本体
  ├── style.css         # スタイル（焦らせる演出の配色・アニメーション）
  ├── app.js            # ロジック（カウントダウン・localStorage・タスク/メモ）
  ├── 仕様書.md
  └── README.md
  ```

### 7.3 公開時の注意
- `localStorage` のデータは**閲覧した端末・ブラウザ内にのみ保存**される。Vercel で公開してもサーバーには保存されず、端末をまたいだ共有・同期はされない。
